# 프로젝트 요구사항 명세서 (PRD)
## Product Requirements Document

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
**스윔 개발자 (Swim Developer Assistant)**

### 1.2 프로젝트 목적
젠스파크(Genspark) 슈퍼에이전트를 이용한 AI 개발 시, 바이브 코딩의 장점은 유지하면서 전문 용어 부족과 반복 작업 문제를 해결하는 **AI 개발 커뮤니케이션 헬퍼 웹 애플리케이션**

### 1.3 배경 및 문제점

#### **기존 문제:**
```
❌ 전문 용어 부족
   - 로고, 버튼, 아이콘, 패널, LNB, 탑 네비게이션 등
   - 용어를 몰라 소통 어려움
   
❌ 번거로운 작업 프로세스
   - 미로(Miro) 웹칠판에 스크린샷
   - 1, 2, 3, 4번 번호 매기기
   - 채팅창에 첨부 및 설명 작성
   - 대기 시간 발생
   
❌ 부분 해결 문제
   - 1~4번 중 일부만 해결
   - 나머지 미해결로 반복 작업
   
❌ 확인 과정의 번거로움
   - 미리보기 창 별도 열기
   - 변경사항 확인 반복
   
❌ 에이전트의 자의적 해석
   - 작업 중 다른 문제 발견 시 임의 수정
   - 누적 오류 발생
```

#### **해결 방향:**
```
✅ 클릭으로 위치 자동 표기
✅ 미리보기와 입력창 통합 화면
✅ Gemini Pro 2.0으로 텍스트 정리
✅ 히스토리 자동 기록 (요구사항 정의서)
✅ GitHub 연동 (실시간 동기화)
```

### 1.4 타겟 사용자
- **주 사용자**: AI 개발 도구(젠스파크 등)를 사용하는 개발자
- **협업 대상**: 3명 이상 개발자가 한 프로젝트를 공동 개발하는 팀
- **사용 환경**: PC 데스크톱 (Chrome 브라우저)

### 1.5 핵심 가치
```
1. 시간 절약: 스크린샷 작업 → 클릭 한 번
2. 정확한 소통: [data-component] 위치 정보 자동 생성
3. 반복 작업 제거: 히스토리 기록으로 재사용
4. 협업 효율: GitHub 연동으로 실시간 동기화
```

## 2. 핵심 기능

### 2.1 미리보기 패널 (화면 좌측 60%)

#### **2.1.1 기본 기능**
- **화면 분할**: 좌측 60%, 우측 40% (드래그로 조절 가능)
- **URL 로딩**: 설정에서 입력한 개발 페이지 URL 표시
- **iframe 렌더링**: 실제 웹페이지를 iframe으로 표시

#### **2.1.2 클릭 모드 (기본)**
```
동작:
1. 사용자가 미리보기 화면의 요소 클릭
2. 해당 요소의 [data-component] 속성 추출
3. 우측 입력창에 위치 정보 자동 입력
   예: [data-component="button"][data-variant="primary"]

특징:
- 즉시 반응 (딜레이 없음)
- 여러 요소 클릭 가능 (누적)
- 각 위치 정보마다 [×] 삭제 버튼
```

#### **2.1.3 스크린 모드 (0.7초 롱프레스)**
```
동작:
1. 사용자가 요소를 0.7초 이상 누름
2. 자동으로 스크린 모드 전환
3. 드래그로 사각형 영역 선택
4. 선택 완료 시 위치 정보만 입력창에 전달
   (캡처 이미지는 생성 안 함)

특징:
- 여러 요소를 한 번에 선택 가능
- 위치 정보만 텍스트로 전달
```

#### **2.1.4 상단 바**
```
[좌측 정렬]
- [클릭모드] 버튼 (기본 활성)
- [스크린모드] 버튼

[우측 정렬]
- [새로고침] 버튼
  - 기본: iframe 새로고침
  - GitHub 연동 시: GitHub에서 최신 버전 pull 후 새로고침
```

```
   - 버튼 텍스트 → [초기화]
   
2. [초기화] 클릭
   - 상단 입력창 내용 유지 (삭제 안 함)
   - 이유: 변경되지 않은 내용 확인용
   - 버튼 텍스트 → [엔터]
```

```
- 채팅 기록 저장 (로컬 스토리지)

  1. 클립보드에 텍스트 복사
  2. 첨부 이미지 포함
  3. Gemini가 히스토리 프롬프트 기반으로 분석
  4. 히스토리 패널에 자동 기록

연결 상태 표시:
- ●(녹색): 연결됨
- ●(빨간색): 연결 안 됨

표시 항목:

- ●협업: [협업 GitHub URL]
```

### 2.4 히스토리 패널

#### **2.4.1 전환 동작**
```
탑 네비게이션 [히스토리] 클릭:
- 좌측 미리보기 패널 → 히스토리 패널로 전환
- 버튼 텍스트: 히스토리 → 미리보기

[미리보기] 클릭:
- 좌측 히스토리 패널 → 미리보기 패널로 전환
- 버튼 텍스트: 미리보기 → 히스토리
```

#### **2.4.2 히스토리 구조**
```
형식:
날짜_[위치정보]_기능설명

예시:
2025.10.30_[Header > btn.primary]_버튼 배경색 변경 (#000 → #2196F3)

상세:
날짜: 2025.10.30
위치정보: [Header > btn.primary]
기능설명: 버튼 배경색 변경 (#000 → #2196F3)
카테고리: 스타일 (자동 분류 - 히스토리에 미포함)
```

#### **2.4.3 히스토리 자동 생성**
```
트리거:
하단 채팅창 [복사] 버튼 클릭

프로세스:
1. 사용자가 복사 버튼 클릭
2. Gemini가 채팅 내용 분석
3. 히스토리 프롬프트 기반으로 판단:
   - 디자인 변경: 히스토리에 기록 안 함
   - 위치 변경: 히스토리에 기록 안 함
   - 기능 변경: 히스토리에 기록
   - 사용자 의견/요청: 히스토리에 기록
4. 히스토리 패널에 자동 추가

목적:
- 요구사항 정의서 자동 생성
- 각 위치별로 정리
- 외주 개발 시 제공
```

#### **2.4.4 히스토리 기능**
```
필터:
- 날짜 (오름차순/내림차순)
- 위치 (특정 위치만 보기)
- 키워드 검색

내보내기:
- Markdown (.md) 형식
- 요구사항 정의서 형식

양식:
날짜: YYYY.MM.DD
위치: [경로 > 요소]
기능: 상세 설명
```

### 2.5 GitHub 연동

#### **2.5.1 1차 목표 (개발 페이지)**
```
목적:
- 웹페이지 디자인을 통해 기능 개발
- HTML 정보 자동 로드
- 접속 URL 변경 자동 감지

필요 정보:
1. HTML 파일 구조
2. data-* 속성 정보
3. 접속 URL (GitHub Pages)
4. 변경 로그

폴더 구조:
project/
├─ index.html
├─ components/
│  ├─ header.html
│  └─ footer.html
└─ ... (기타 파일)
```

#### **2.5.2 2차 목표 (협업 개발)**
```
목적:
- 3명 이상 개발자 공동 작업
- v1.6 표준문서 구조 연동
- 중복 작업 방지
- 실시간 동기화

필요 정보:
1. docs/ 폴더 (5종 문서)
   - PRD.md
   - API_SPEC.md
   - DATABASE_SCHEMA.md
   - DESIGN_SYSTEM.md
   - PAGE_SPECS.md

2. shared_memory.json
   - last_component_id
   - used_class_names
   - api_endpoints
   - global_state_keys

3. Commit 로그
   - 누가 (개발자)
   - 언제 (시간)
   - 무엇을 (파일)
   - 어떻게 (변경 내용)

4. 작업 잠금 (Lock) 정보
   - 현재 작업 중인 파일/컴포넌트
   - 작업자 정보
   - 예상 완료 시간
```

#### **2.5.3 충돌 방지 메커니즘**
```
시나리오:
PC1 (개발자 A): Header 작업 중
PC2 (개발자 B): Header 작업 시도

동작:
1. 개발자 B가 Header 클릭
2. 웹사이트가 shared_memory.json 확인
3. Header 작업 중임을 감지
4. 경고창 표시:
   "⚠️ 현재 개발자 A가 Header 작업 중입니다.
   예상 완료 시간: 30분 후
   [대기] [다른 작업 선택]"

규칙:
- 로그 파일 기반 중복 작업 감지
- 실시간 잠금 정보 표시
- 작업 완료 후 자동 잠금 해제
```

#### **2.5.4 연결 상태 표시**
```
위치: 하단 채팅창

표시:
●저장소: https://github.com/user/project
●협업: https://github.com/team/project

색상:
- 녹색(●): 연결됨, 동기화 완료
- 빨간색(●): 연결 안 됨, 오류
- 주황색(●): 연결 중, 동기화 중
```

#### **2.5.5 인증 방법**
```
방법: Personal Access Token (PAT)

설정:
1. 설정 > GitHub 탭
2. [GitHub Token 발급 방법 보기] 링크 클릭
   → https://github.com/settings/tokens 새 탭 열기
3. Token 입력 필드
4. [연결 테스트] 버튼
5. 성공 시: ✅ "연결되었습니다"
6. 실패 시: ❌ "Token을 확인하세요"

저장:
- 로컬 스토리지에 암호화 저장
```

## 3. 비기능 요구사항

### 3.1 성능
```
- 미리보기 로딩: 3초 이내
- 클릭 반응 속도: 즉시 (< 100ms)
- Gemini 응답: 5초 이내
- GitHub 동기화: 10초 이내
```

### 3.2 보안
```
- API 키 암호화 저장 (로컬 스토리지)
```

```
- 브라우저: Chrome 최신 버전
- 화면 해상도: 최소 1920x1080
- 플랫폼: Windows, macOS
```

### 3.4 디자인
```
- 스타일: Apple (macOS) 스타일
- 색상: 밝은 회색 배경, 흰색 패널
- 폰트: 시스템 폰트 (SF Pro 또는 Pretendard)
- 아이콘: 심플, 20px
- 간격: 여유로운 패딩
```

### 3.5 접근성
```
- 키보드 네비게이션
- 고대비 모드 지원 (다크 모드)
- 폰트 크기 조절 (3단계)
```

```
✅ 화면 분할 (미리보기 + 입력창)
```
